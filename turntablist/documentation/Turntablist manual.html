Turntablist Manual

 

Welcome to Turntablist 1.1

Turntablist is an Audio Unit instrument plugin that can load an audio sample and perform turntable style effects on it. You can scratch, change pitch, and even spin the audio up and down, just like with a real turntable.

Using Turntablist

Getting results:

  1. Load the plugin in an Audio Unit instrument host.
  2. Open an audio file.
  3. Play a MIDI note or press the Play button.

Scratching:

Use the mouse to quickly move the Scratch Amount slider back and forth while the sample is playing. You can also turn the Power button off, and turn Play on and scratch without the sample having to play on its own. You can also use the pitch bend on your MIDI controller.

Spin down effect:

Turn Note-Power Track off, Power on and put Spin Down to around 50%. While Play button is on, turn the Power off.

Power effects:

Set the Spin Up Speed and Spin Down Speed to different values. Turn Note-Power Track on and turn Power on. Hit the Play button on and off and notice the sampling spinning up, playing and spinning down. This is a nice effect to use on beat loops.

A note about performance:

The scrub scratch mode will vary in response depending on your host software. The scratching is speed based (not position based) and will lose tracking after a while. How quickly it loses tracking depends on the host.

Power controls

 
Open File
  • use this to find an audio file to load up onto the "turntable"
Power
  • switches the turntable power on or off
Spin Up Speed
  • how quickly the audio playback "spins up" when the turntable power turns on
Spin Down Speed
  • how quickly the audio playback "spins down" when the turntable power turns off
Note-Power Track
  • enabling this will cause note-on and note-off messages to be mapped to turntable power on and off for an interesting effect

Play controls

 
Play
  • push this on or off to start or stop the audio sample playback
Note Mode
  • toggles between "reset" and "resume" mode
  • In resume mode, new notes cause the audio sample to continue playing from where it last stopped.
  • In reset mode, new notes cause the audio sample to restart playback from the beginning.
Direction
  • changes playback direction of the audio sample, regular or reverse
Loop
  • if you enable this, the audio sample playback will continuously loop
Key Track
  • switches key tracking on or off
  • When key tracking is on, the pitch and speed of the audio sample playback are transposed in relation to the pitch of the note. The center key (no transposition) defaults to middle C, but can be changed via the Root Key parameter.
  • When key tracking is off, note pitches are ignored.
Root Key
  • when Key Track is enabled, this controls the MIDI key around which the transposition is centered

Pitch and Scratch controls

 
Pitch Range
  • this controls the range of pitch adjustment values that the Pitch Shift parameter offers
Pitch Shift
  • changes the audio playback pitch between +/- the Pitch Range value
Scratch Mode
  • toggles between "scrub" mode and "spin" mode, which affects the behavior of the Scratch Amount parameter
  • Scrub mode is like when you move a record back and forth quickly with your hand.
  • Spin mode is like when you speed up or slow down the record spinning with your hand.
Scratch Speed
  • sets the speed of the scratching effect that the Scratch Amount parameter produces
  • In scrub mode, the range of Scratch Speed represents time from 0.5 to 5 seconds.
  • In spin mode, the range of Scratch Speed represents the maximum rate from 1 to 8 times the playback rate.
Scratch Amount
  • This slider is what does the actual scratching.
  • In scrub mode, the entire range of the slider represents time (set by the Scratch Speed parameter).
  • In spin mode, the top and bottom of the slider represent the maximum forward and backward speed (set by the Scratch Speed parameter) while the center is 0.
  • If the Power and Play buttons are on and the Scratch Amount slider is in the center, scratching is disabled and the audio sample just plays back normally.
  • You can optionally use MIDI pitchbend to control Scratch Amount.
 

Helper controls

 
MIDI Learn
  • switches MIDI Learn mode on or off
  • When MIDI Learn is on, you can click on a parameter control to enable that parameter as the "learner." Then the next MIDI CC message that is received will be mapped to that parameter. From then on, you can control that parameter with MIDI CC messages of that type.
  • These MIDI CC → parameter assignments are saved with the plugin's settings.
MIDI Reset
  • removes all of your MIDI CC → parameter assignments
Help
  • push this to view the manual that you are reading right now

Version history:

1.1.0 - June 18, 2004 - 2005

  • First Audio Unit release, by Sophia Poirier of Destroy FX. Bioroid stopped development and released the source code, and Sophia decided to tinker with it.
  • now released under the terms of the GNU General Public License
  • fixed numerous bugs in the audio file handling, which previously would not work in many cases (if files were located on non-boot volumes, if file paths were longer than 1024 bytes, if file names had Unicode characters, if file names were longer than 31 characters, etc.)
  • fixed a bug in the resampling interpolation algorithm so that the sound quality is much better now
  • fixed a buffer overrun bug in the audio playback that could cause audio corruption or crashing
  • no longer requires that you install the libsndfile.dylib library
  • added "MIDI Learn" feature
  • you can drag'n'drop™ audio files and preset files onto the plugin GUI to load them
  • the manual is now built into the plugin
  • each control has pop-up help that appears when you hold your mouse over a control
  • mouse wheel support added for adjusting any controls
  • fixed a bug where the Pitch Shift parameter value used was miscalculated and smaller than what you saw (and I also vastly increased the Pitch Range maximum)
  • the audio file loading and audio rendering are now synchronized so as to avoid misbehavior (like crashing)
  • fixed rounding/resolution errors in the varispeed playback
  • fixed the root key off-by-2 bug, and made the root key a controllable parameter
  • when an audio file loaded in a previous session cannot be found, you get a message informing you of that

  • fixed: libsndfile loading issue in Plogue Bidule on Mac OS X
